## Metaform

Metaform - открытая библиотека для работы с API [Анилибрии](https://github.com/anilibria), написанная на [TypeScript](https://www.typescriptlang.org/) и распространяющаяся под открытой лицензией [MIT](https://ru.wikipedia.org/wiki/Лицензия_MIT). Является следующим этапом развития идей заложенных в [XConnect](https://github.com/maxqwars/xconnect) и исправлением ошибок допущенных при разработке последнего. Написанная с использованием современных стандартов ECMAScript и API основанным на ES Promises.

### Что такое XConnect

XConnect - первая проба пера по написанию библиотеки для работы с API Анилибрии, заглохшая по причине накопившихся при её написании проблем.

XConnect имел хорошие идеи, но плохую реализацию этих идей и в конце концов бы похоронен, уступив место Metaform.

#### С чего всё началось

Вся история начинается в начале октября 2021 года, когда мне в голову пришла идея ставшая затем головной болью, написать свой клиент для AniLibria используя открытый API. Проект чисто для себя, развить навыки использования React и JavaScript, что бы потом сие творение оказалось в моём портфолио разработчика.

Изначально в основе нового приложения лежал [React Native](https://reactnative.dev/) и горстка костылей, которая позволила бы собирать проект не только как мобильное приложение для android и ios, но и разворачивать его в web. Впрочем, это отдельная история, история Pathogen и его предшественника Sayan.

Прошерстив GitHub было найдено много проектов уже добившихся определенного успеха в этом деле, и первая проблема возникшая при исследовании этих проектов, осознание того что логика работы с API вшита в сам проект, а не оформлена в виде отдельной библиотеки, так было принято решение разделить весь проект на два независимых Sayan и XConnect.

XConnect был проектом целью которого стала работа с API и предоставление набора удобных классов для работы с ним, но об этом подробнее в части про архитектуру XConnect. XConnect должен был стать не только основой для Sayan, но и для других проектов использующих это API.

## Архитектура XConnect

Metaform - вобрала в себя все лучшие идеи заложенные в XConnect, что бы избавится от недостатков.

В Metaform присутствует 3 набора сущностей доступных для импорта.

### Core

Базовые классы необходимые для выполнения базовых операций работы с API, в принципе для работы с API их вполне достаточно, однако они не такие удобные как модули. Сюда входит построитель url и много QueryBuilder(ов)

### Modules

Основные классы Metaform, используют корневые классы в своей основе и предоставляют удобный интерфейс для работы с API.

- **Database** модуль работы с релизами

- **Search** модуль работы с поиском

- **Updates** модуль работы с обновлениями контента

### Function, Utils

Функции и классы которые могут и не понадобится вам при работе с API.

Однако часть из этих функций используется в модулях.

## Metaform VS XConnect

Небольшое сравнение XConnect и Metaform

| Feature             | Metaform | XConnect |
| ------------------- | -------- | -------- |
| TypeScript          | Y        | Y        |
| MIT                 | Y        | Y        |
| semantic versioning | Y        | Y        |
| CI/CD               | Y        | N        |
| npm                 | Y        | N        |
| Unit testing        | Y        | N        |
| Timeout fetch       | Y        | N        |
| API Error handing   | Y        | N        |
| Cross-platform      | Y        | Y        |
| Modules             | Y        | Y        |
| Core classes        | Y        | Y        |
